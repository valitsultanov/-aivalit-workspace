# MASTER HOST INFO — единый центр контекста проектов

Цель: чтобы любой чат/сессия OpenClaw понимали состояние проектов одинаково.

## Как теперь работаем

1. Валит записывает аудио по проекту.
2. Аудио кладётся в папку проекта: `projects/<project>/audio/`.
3. Краткая расшифровка/суть кладётся в `projects/<project>/notes/`.
4. Из заметок формируется понятный статус и задачи в `projects/<project>/tasks/`.
5. Ключевые выводы дублируются в `projects/_master-host-info/_summaries/`.

## Стандарт папки проекта

В каждом проекте:
- `audio/` — голосовые брифы
- `inbox/` — сырые материалы
- `notes/` — структурированные заметки
- `tasks/` — план действий и next steps

## Единые правила

- Один аудиофайл = одна тема/сессия.
- Именование аудио: `YYYY-MM-DD_HH-mm_topic.m4a`.
- На каждый аудиофайл — мини-итог в `notes/` (3 блока):
  - Контекст
  - Решения
  - Что делаем дальше
- В `tasks/` держать только конкретные действия с дедлайном/приоритетом.

## Быстрый шаблон заметки

```md
# <project> — audio brief — YYYY-MM-DD

## Контекст

## Решения

## Следующие шаги
- [ ] ...
```

## Важно

Это «главный хост инфы»: сюда сводится всё, чтобы даже из main-чата быстро восстановить картину по каждому проекту.
