# Git Workflow

## Ветки
- `main` — стабильная
- `dev` — рабочая

## Поток работы
1. Работаем в `dev`
2. Коммитим атомарно (`feat/fix/docs/chore`)
3. Push в `origin/dev`
4. PR: `dev -> main`
5. Merge после проверки

## Запреты
- Не хранить токены/ключи в репо
- Не пушить файлы >100MB
- Не работать напрямую в `main` для крупных изменений
