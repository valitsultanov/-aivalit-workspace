## 2) Как использовать базу (операционный порядок)

### Шаг 1 — Входящий запрос
Открываем `01_Onboarding`:
- приветствие,
- бриф 6Q,
- маршрутизация.

### Шаг 2 — Выбор решения
Открываем `02_Products`:
- выбираем 1 основной + 1 запасной формат,
- отправляем не более 3 ссылок.

### Шаг 3 — Усиление доверием
Открываем `03_Cases`:
- даём кейс по сегменту,
- закрываем сомнение,
- ведём к CTA.

### Шаг 4 — Выдача материалов
По сценарию подключаем:
- `04_Guides`
- `05_Video`
- `06_Presentations`
- `07_PDF`

### Шаг 5 — Быстрые ответы
Открываем `08_FAQ` для частых вопросов.

### Шаг 6 — Коммуникация в Telegram
Открываем `09_Telegram_Assets`:
- закрепы,
- шаблоны,
- CTA-цепочки.

## 3) Универсальные правила выдачи

1. Не больше 3 материалов за раз.
2. Каждый ответ заканчивается следующим шагом (CTA).
3. Один ответ = одна цель.
4. Не обещать невозможного.
5. Для новичка: сначала Quick-материалы.

## 4) Сегменты и стартовые маршруты

### Новичок
- Onboarding → Guides (Quick) → 1 Video (Quick) → CTA “7-дневный план”.

### Маркетолог / креатор
- Onboarding → Products → Cases (контент) → Video (Core) → CTA “внедрение”.

### Предприниматель / эксперт
- Onboarding → Products (ROI-фокус) → Cases (бизнес) → PDF/Core Deck → CTA “разбор”.

### Запрос по SV SPOT
- Onboarding → SV SPOT сценарий → кейс → презентация/гайд → CTA “персональный разбор”.
