# AiValit PDF — All In One

## 0) Цель
Собрать PDF-библиотеку так, чтобы:
- быстро находить нужный документ,
- выдавать PDF по задаче и уровню клиента,
- не перегружать пользователя,
- фиксировать следующий шаг после чтения.

---

## 1) Стандарты PDF

**Формат:**
- PDF (основной формат выдачи)
- Желательно: размер до 15–20 МБ для быстрой отправки

**Рекомендуемый объём:**
- Quick PDF: 2–8 стр.
- Core PDF: 8–20 стр.
- Deep PDF: 20–60 стр.

**Нейминг файла:**
ТЕМА__СЕГМЕНТ__ТИП__YYYY-MM-DD__v1.pdf

Пример:
AI_Onboarding__Beginner__QuickPDF__2026-02-23__v1.pdf

---

## 2) Карточка PDF (метаданные)

### [PDF_NAME]
**Сегмент:** [новичок / маркетолог / предприниматель / эксперт / команда]  
**Тип:** Quick / Core / Deep  
**Задача документа:** [что решает]  
**Кратко что внутри (3–5 пунктов):**
- [пункт 1]
- [пункт 2]
- [пункт 3]

**Объём:** [X страниц]  
**Ссылка:** [URL]  
**Когда выдавать:** [сценарий]  
**CTA после чтения:** [следующий шаг]

---

## 4) Логика выдачи PDF

### Этап 1 — Первый контакт
- Даем 1 Quick PDF
- Цель: дать ясность без перегруза

### Этап 2 — Уточнение выбора
- Даем 1 Core PDF + 1 доп. материал (по запросу)
- Цель: помочь принять решение

### Этап 3 — Внедрение
- Даем 1 Deep PDF (только тем, кто готов внедрять)
- Цель: перейти к системной работе

---

## 5) Шаблоны выдачи PDF в Telegram

### 5.1 Короткая выдача
Под твою задачу рекомендую этот PDF: **[название]**  
Объём: [X стр], формат: [Quick/Core/Deep]  
Ссылка: [URL]  
Прочитай и напиши “готово” — дам следующий шаг.

### 5.2 Пакет до 2 PDF
Собрал 2 PDF по твоей задаче:
1) [Название + ссылка] — [зачем]
2) [Название + ссылка] — [зачем]

Начни с первого. Если откликается — соберу персональный план внедрения.

### 5.3 Если пользователь перегружен
Чтобы не перегружать, начни с одного короткого PDF:
**[Название]** — [ссылка]  
Это даст базовую ясность за 10–15 минут.

---

## 6) Правила порядка
- Не отправлять больше 2 PDF за раз
- На каждый PDF должен быть сценарий “когда выдавать”
- На каждый PDF должен быть CTA
- Версионирование: `v2`, `v3` при обновлении
- Устаревшие версии переносить в `99_Archive`

---

## 7) Контрольный чеклист
Перед добавлением PDF:

- [ ] Есть рабочая ссылка
- [ ] Есть объём (страницы)
