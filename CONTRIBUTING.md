# Contributing

## Branching
- `main` — стабильная ветка
- `dev` — рабочая (рекомендуется)

## Commit Convention
Используем краткие префиксы:
- `feat:` новая функциональность
- `fix:` исправление
- `docs:` документация/контент
- `chore:` сервисные изменения
- `refactor:` улучшение без изменения поведения

Пример:
`docs(content): update onboarding flow v2`

## Pull Requests
1. Ветка от `dev`
2. Маленькие атомарные изменения
3. Понятное описание: что/зачем/риски
4. После ревью — merge

## Safety
- Не коммитить токены, `.env`, ключи
- Крупные бинарники >100MB не хранить в git (или Git LFS)
